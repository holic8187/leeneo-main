# V2 스킬 해금 퀘스트 및 마스터리북 구현 계획

> 상태: **검수안 기준 구현 완료**  
> 기준 문서: `QUEST_UNLOCK_SKILLS.md`  
> 이 문서의 퀘스트 흐름, 단계별 투자 상한, 마스터리북 규칙은 현재 V2 서버에 적용되어 있다.

## 1. 확정 규칙

1. `quest: true`인 스킬은 처음에는 투자할 수 없다.
2. 기본 해금 퀘스트를 완료하면 스킬을 배우고 최대 10레벨까지 투자할 수 있다.
3. 마스터리북 대상이 아닌 스킬은 연계 퀘스트를 완료할 때마다 상한이 `10 → 20 → 30`으로 열린다.
4. 최대 레벨이 20인 스킬은 `10 → 20`, 최대 레벨이 25인 스킬은 `10 → 20 → 25`로 처리한다.
5. 마스터리북 대상 스킬은 연계 퀘스트 대신 해당 스킬의 20·30 마스터리북으로 상한을 연다.
6. 마스터리북 20의 기본 성공 확률은 90%, 마스터리북 30의 기본 성공 확률은 70%다.
7. 실패할 때마다 같은 캐릭터·같은 스킬·같은 단계의 다음 시도 성공 확률이 1%p씩 누적 증가한다.
8. 마스터리북은 성공과 실패 모두 1개 소모한다. 성공하면 해당 단계의 실패 누적치는 0으로 초기화한다.
9. 마스터리북을 드롭할 몬스터와 개별 드롭률은 추후 지정한다. 먼저 아이템과 사용 로직만 만들 수 있게 틀을 준비한다.
10. 스킬 해금 퀘스트는 계정당 1회 완료한다. 부서를 변경했다가 돌아와도 이미 달성한 해금 단계는 유지한다.

## 2. 해금 방식

### 2.1 퀘스트 전용 스킬

```text
잠김
  ↓ 기본 해금 퀘스트
최대 10레벨
  ↓ 1차 연계 퀘스트
최대 20레벨
  ↓ 2차 연계 퀘스트
최대 30레벨
```

### 2.2 퀘스트 + 마스터리북 스킬

```text
잠김
  ↓ 기본 해금 퀘스트
최대 10레벨
  ↓ 마스터리북 20 성공
최대 20레벨
  ↓ 마스터리북 30 성공
최대 30레벨
```

### 2.3 마스터리북 전용 스킬

`quest: false`이지만 사용자 문서에서 마스터리북 대상으로 지정한 스킬이다. 별도 기본 퀘스트 없이 처음부터 10레벨까지 투자할 수 있고, 이후 마스터리북으로 상한을 연다.

### 2.4 예외 상한

| 스킬 | 제안 상한 흐름 | 비고 |
|---|---:|---|
| 전 직원 역량강화 | 기본 10 → 마스터리북 20 | 현재 코드상 최대 30이므로 실제 구현 시 최대 레벨을 20으로 맞춘다. |
| 속성 부여: 성 | 퀘스트 10 → 연계 퀘스트 20 | 원래 최대 레벨 20 |
| 동반자: 회복 | 퀘스트 10 → 연계 퀘스트 25 | 원래 최대 레벨 25 |
| 동반자: 버프 | 퀘스트 10 → 연계 퀘스트 25 | 원래 최대 레벨 25 |
| 복직 지원 | 기본 해금 퀘스트 → 최대 10 | 현재 최대 레벨 10이라 후속 단계 없음 |

## 3. 저장 구조 제안

현재 코드는 `skills.unlockedQuestSkills`에 스킬 ID만 저장한다. 이 방식으로는 10·20·30 상한과 실패 보정을 구분할 수 없으므로 다음처럼 부서별 진행 상태를 추가한다.

```js
skills.unlockProgress = {
  accounting: {
    extended_2926a732db: {
      cap: 20,
      unlockedAt: '2026-07-14T00:00:00.000Z',
      failures: { 20: 0, 30: 2 }
    }
  }
};
```

- 진행 상태는 `부서 ID → 스킬 ID` 순으로 분리한다.
- 공용 스킬도 부서별 진행 상태를 따로 가진다. 이렇게 해야 같은 `정신 차려!`라도 부서별 해금 방식이 달라도 충돌하지 않는다.
- 직업 변경권을 사용해도 이전 부서의 해금 기록은 삭제하지 않는다.
- 현재 투자 레벨은 기존 `skills.levels`를 그대로 사용한다.
- 상한보다 높은 레벨이 이미 투자되어 있다면 절대 포인트를 회수하거나 레벨을 낮추지 않는다.

## 4. 기존 계정 이관 원칙

기존에 이미 해금되었거나 투자된 스킬이 새 시스템 때문에 잠기지 않도록 다음 원칙으로 1회 이관한다.

| 현재 상태 | 이관 상한 |
|---|---:|
| 해금 기록은 있으나 투자 0~10 | 10 |
| 현재 투자 11~20 | 20 |
| 현재 투자 21 이상 | 해당 스킬의 원래 최대 레벨 |
| 마스터리북 전용 스킬에 이미 11 이상 투자 | 현재 투자 레벨을 포함하는 단계까지 자동 해금 |

- 기존 `unlockedQuestSkills`는 호환용으로 당분간 유지하고, 새 구조가 정상 이관된 뒤 읽기 전용으로 취급한다.
- 이관은 여러 번 실행해도 중복 보상이나 단계 하락이 없는 방식으로 만든다.
- 이미 완료한 일반 NPC 퀘스트는 변경하지 않는다.

## 5. 퀘스트 목표 확장안

현재 퀘스트 엔진의 `처치`, `수집`, `맵 방문`, `보스 처치`를 그대로 활용하되, 스킬의 성격을 살리기 위해 아래 목표를 추가한다.

| 목표 유형 | 예시 | 서버 판정 |
|---|---|---|
| 스킬 사용 | 콤보어택 50회 사용 | 유효한 스킬 시전 완료 시 누적 |
| 누적 피해 | 특정 계열 스킬로 100만 피해 | 실제 적용 피해만 누적 |
| 피격·생존 | 사망하지 않고 100회 피격 | 무효화된 공격은 별도 조건으로 구분 |
| 파티 지원 | 파티원에게 버프·회복 50회 | 자신만 대상인 사용은 제외 가능 |
| 속성 처치 | 불·얼음·번개·성 공격으로 100마리 처치 | 막타 속성 기준 |
| 제한 시간 | 지정 몬스터를 3분 안에 처치 | 수락 후 별도 도전 시작 버튼 제공 |
| 무사망 보스 | 보스 처치까지 사망하지 않기 | 같은 맵·기여 조건 확인 |

기본 해금 퀘스트는 아직 해당 스킬을 쓸 수 없으므로 `스킬 사용` 조건을 넣지 않는다. 스킬 사용 조건은 20·30 상한 연계 퀘스트에만 배치한다.

NPC 이름과 맵은 현재 NPC·맵 목록을 우선 재사용한다. 아래 제목과 수량은 검수 후 조정한다.

### 6.1 공통

| 스킬 | 방식 | 퀘스트 흐름 제안 | 담당 NPC / 맵 |

| 정신 차려! | 부서별 해금 | `임원 호출`(임원 전략회의층 방문) → `정신력 검증`(무사망 처치) → `끝나지 않은 회의`(황과장 기여 처치) | 출근 도장을 잃어버린 피치 / 중앙로비 |

전직원 역량 강화-> 기본 해금된 상태로 만렙 10으로 설정. 스킬마스터리북 20을 통해 만렙 20까지 확장, 20렙 기준 600초간 파티원 기본 스탯 10% 증가.

## 8. 마스터리북 아이템 규칙

- 아이템명: `<스킬명> 마스터리북 20`, `<스킬명> 마스터리북 30`
- 분류: 소비 아이템
- 중첩: 한 칸에 최대 1개
- 사용 조건:
  - 교환 가능, 거래 가능
  - 현재 부서에서 배울 수 있는 스킬일 것
  - 퀘스트 스킬은 기본 해금 퀘스트를 완료했을 것
  - 20북은 현재 상한이 10이며 해당 스킬을 5 이상 찍었을것
  - 30북은 현재 상한이 20이며 해당 스킬을 15 이상 찍었을것
- 이미 열린 단계의 책을 사용하면 소모하지 않고 안내한다.
- 성공 메시지: `마스터리북의 지식이 스킬에 완전히 전해졌습니다. 이제 <스킬명>을(를) 최대 <단계>레벨까지 올릴 수 있습니다.`
- 실패 메시지: `마스터리북의 지식이 흩어졌습니다. 다음 성공 확률이 <확률>%로 상승했습니다.`
- 인벤토리 설명에 현재 기본 확률과 실패 보정 규칙을 표시한다.
- 상점 판매가 1원

## 9. 화면 구성

### 스킬 창

- 잠긴 스킬에는 자물쇠와 `해금 퀘스트 필요`를 표시한다.
- 해금된 스킬에는 `현재 투자 상한 10/20/30`을 표시한다.
- 상한에 도달하면 `연계 퀘스트 필요` 또는 `마스터리북 20 필요`를 구분해 표시한다.
- 해당 퀘스트를 받을 수 있으면 스킬 상세에 `퀘스트 위치 보기` 버튼을 제공한다.

### 퀘스트 창

- 스킬 해금 퀘스트는 별도 `스킬` 필터로 모아볼 수 있게 한다.
- 연계 퀘스트는 이전 단계를 완료하기 전에는 목록에 노출하지 않는다.
- 보상 미리보기는 기존 규칙대로 완료 가능 상태가 되기 전까지 숨긴다.

### 마스터리북 사용

- 사용 전 현재 성공 확률을 내부 확인창에 표시한다.
- 실패 시 다음 확률을 즉시 표시한다.
- 모바일에서도 별도 브라우저 알림이 아닌 사이트 내부 결과창을 사용한다.

## 10. 구현 순서

1. 스킬별 상한·실패 누적 데이터와 기존 계정 이관 추가

5. 스킬 창·퀘스트 창·마스터리북 결과 UI 추가
6. 부서별 기본·연계 퀘스트 등록

## 11. 필수 테스트

- 기본 퀘스트 전에는 스킬 포인트를 투자할 수 없다.
- 기본 퀘스트 후 10레벨까지만 투자된다.
- 연계 퀘스트 또는 마스터리북 성공 후에만 다음 단계 투자가 가능하다.
- 20북·30북 실패 시 책이 1개 소모되고 성공 확률이 정확히 1%p 증가한다.
- 성공 시 실패 누적치가 초기화되고 재사용으로 책이 낭비되지 않는다.
- 직업 변경 후 다른 부서 진행도와 기존 부서 진행도가 섞이지 않는다.
- 기존 고레벨 캐릭터의 투자 스킬 레벨과 스킬 포인트가 줄어들지 않는다.
- 계정당 1회 퀘스트가 재접속·서버 재시작 후에도 중복 완료되지 않는다.
- 25레벨 스킬이 20에서 막히거나 30까지 잘못 열리지 않는다.

## 12. 사용자 검수 필요 항목

1. `정신 차려!`의 현재 최대 레벨은 10이다. 회계팀만 20북을 쓰게 할지, 모든 부서의 최대 레벨을 20 또는 30으로 늘릴지 확정이 필요하다. -> 10으로 설정
2. `전 직원 역량강화`는 현재 최대 30이지만 사용자 원문대로 최대 20으로 낮추는 안으로 작성했다. -수정완료
3. 마스터리북의 교환 가능 여부와 상점 판매가를 정해야 한다. - 수정완료
4. 퀘스트 목표 수량과 보상 경험치·아이템은 아직 미정이다. - 수정완료
5. 황과장 처치가 필요한 퀘스트는 필드보스 대기 시간이 길 수 있으므로 대체 목표를 허용할지 결정해야 한다. - 기존 그대로, 대체목표 불허
6. 마스터리북 사용에 `현재 스킬 레벨 5/15 이상` 같은 추가 조건은 넣지 않은 상태다. - 수정완료
