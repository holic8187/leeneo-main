# 기존 Render 주소를 V2로 전환하기

현재 기본값은 `APP_MODE=v1`입니다. 따라서 코드를 먼저 배포해도 V1은 그대로 열립니다.

## 최종 전환

1. Render 서비스의 `Environment` 메뉴를 엽니다.
2. 환경변수 `APP_MODE`를 추가하고 값을 `v2`로 설정합니다.
3. 기존 `MONGO_URI`와 `JWT_SECRET`은 그대로 유지합니다.
4. 저장 후 재배포합니다.
5. 기존 Render 주소에 접속하면 `/v2/`로 자동 이동하는지 확인합니다.
6. `/api/health` 응답의 `appMode`가 `v2`인지 확인합니다.

## V2 모드에서 달라지는 점

- `/`와 `/index.html`은 `/v2/`로 이동합니다.
- `/api/v2/*`와 `/api/health`만 사용할 수 있습니다.
- V1 API는 HTTP 410 응답으로 닫힙니다.
- V1 주간 개인면담 정산 작업은 실행되지 않습니다.
- 기존 V1 MongoDB 문서는 삭제하지 않습니다.
- V2 계정·스냅샷·캐릭터는 별도 `v2_*` 컬렉션에 저장됩니다.

## 되돌리기

긴급 롤백이 필요하면 `APP_MODE`를 `v1`으로 되돌리고 재배포합니다. V1 원본 데이터는 삭제하지 않았기 때문에 즉시 복귀할 수 있습니다.
